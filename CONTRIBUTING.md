# Contributing to GridSenpAI

Thanks for your interest in contributing to GridSenpAI.

## Ground rules
- Be respectful and constructive.
- Keep discussions technical and evidence-based.
- Do not share proprietary or sensitive infrastructure information (e.g., customer one-lines, relay settings, substation details) unless you have explicit permission and the project maintainers request it.

## How to contribute
1. Fork the repo and create a feature branch.
2. Make your changes with clear commits.
3. Add or update documentation and tests where applicable.
4. Open a Pull Request (PR) and describe what you changed, why you changed it, and how to test it.

## Issues
- Use Issues to propose features, report bugs, or discuss research questions.
- Include reproduction steps for bugs and attach minimal, anonymized examples.

## Coding standards
- Prefer simple, readable code.
- Avoid vendor-specific assumptions unless explicitly documented.
- Any AI-generated parameters must be traceable to evidence or clearly marked as assumptions.
